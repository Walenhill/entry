# Система записи на услуги (MVP)

Простой прототип системы бронирования для проверки идеи. Без лишних сложностей — только базовый функционал.

## Что умеет

**Для клиентов:**
- Посмотреть свободные слоты
- Забронировать слот (имя + телефон)

**Для администратора:**
- Войти по паролю (авторизация на основе PHP сессий)
- Создать слот вручную
- Сгенерировать слоты по шаблону (например, каждый час с 9 до 18)
- Видеть все брони и статистику
- Отменять/удалять слоты

## Технологии

- **Бэкенд:** PHP 8.2 (чистый, без фреймворков) + MySQL
- **Фронтенд:** Vue 3 + Vite
- **Инфраструктура:** Docker

## Как запустить

### Вариант 1: Docker (рекомендуется)

```bash
# 1. Скопируйте конфиги
cp backend/.env.example backend/.env
cp fronted-vue/.env.example fronted-vue/.env

# 2. Отредактируйте пароли в backend/.env

# 3. Запустите
docker compose up -d --build
```

Приложение доступно:
- Фронтенд: http://localhost:5173
- Бэкенд API: http://localhost:8080

### Вариант 2: Без Docker

**Бэкенд:**
```bash
cd backend
# Убедитесь, что .env файл настроен для вашей локальной БД
php -S localhost:8080 main.php
```

**Фронтенд:**
```bash
cd fronted-vue
pnpm install
pnpm dev
```

## API Documentation

| Метод | Эндпоинт | Роль | Описание |
|-------|----------|------|----------|
| GET | `/slots` | Любой/Админ | Получение списка слотов. Клиенты видят только свободные, администраторы — все. |
| GET | `/slots/{id}` | Любой/Админ | Получение информации о конкретном слоте. Персональные данные клиента скрыты, если запрос не от администратора. |
| POST | `/slots/{id}/book` | Клиент | Бронирование свободного слота (требуется имя и телефон). |
| POST | `/auth/login` | - | Авторизация администратора. |
| POST | `/auth/logout` | Админ | Выход администратора (уничтожение сессии). |
| GET | `/stats` | Админ | Получение статистики по бронированиям. |
| POST | `/slots` | Админ | Создание нового одиночного слота. |
| POST | `/slots/generate`| Админ | Массовая генерация слотов по шаблону. |
| PUT | `/slots/{id}` | Админ | Обновление слота (в настоящий момент доступно только изменение описания). |
| DELETE| `/slots/{id}` | Админ | Удаление слота. |
| POST | `/slots/{id}/cancel`| Админ | Отмена существующего бронирования (перевод в статус "свободен" и очистка данных клиента). |

## Переменные окружения

**backend/.env:**
- `DB_HOST` - хост MySQL (по умолчанию mysql)
- `DB_PORT` - порт MySQL (по умолчанию 3306)
- `DB_DATABASE` - имя БД
- `DB_USERNAME` - пользователь БД
- `DB_PASSWORD` - пароль БД
- `MASTER_PASSWORD` - пароль администратора (хэшируется при первом входе)
- `APP_ENV` - окружение (например, development)
- `APP_DEBUG` - режим отладки (true/false)
- `FRONTEND_URL` - URL фронтенда для CORS (по умолчанию не задано, используются localhost:5173 и 127.0.0.1:5173)

**fronted-vue/.env:**
- `VITE_API_BASE_URL` - URL бэкенда (по умолчанию http://localhost:8080)

## Известные ограничения

- Отсутствует стандартный тестовый фреймворк (например, PHPUnit). Юнит-тесты реализованы в виде отдельных PHP-скриптов в директории `backend/tests/`.
- Backend API не использует и не требует проверки CSRF-токенов.
- Состояние авторизации на фронтенде опирается на флаг `is_logged_in` в localStorage.
- CORS строго ограничен в `backend/main.php` (разрешены только конкретные домены, например `http://localhost:5173`).

## Безопасность

- Пароли хэшируются через Argon2id и хранятся в БД.
- Сессионная авторизация с защищенными куки (HttpOnly, SameSite).
- Rate limiting: блокировка IP после 5 неудачных попыток на 15 минут.
- Валидация входных данных и защита от пересечения слотов по времени.
