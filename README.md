# Система записи на услуги (MVP)

Простой прототип системы бронирования для проверки идеи. Без лишних сложностей — только базовый функционал.

## Что умеет

**Для клиентов:**
- Посмотреть свободные слоты
- Забронировать слот (имя + телефон)

**Для администратора:**
- Войти по паролю (авторизация на основе PHP сессий)
- Создать слот вручную
- Сгенерировать слоты по шаблону (например, каждый час с 9 до 18)
- Видеть все брони и статистику
- Отменять/удалять слоты

## Технологии

- **Бэкенд:** PHP 8.2 (чистый, без фреймворков) + MySQL
- **Фронтенд:** Vue 3 + Vite, Vue Router, Pinia, PWA (Promis)
- **Инфраструктура:** Docker

## Как запустить

### Вариант 1: Docker (рекомендуется)

```bash
# 1. Скопируйте конфиги
cp backend/.env.example backend/.env
cp fronted-vue/.env.example fronted-vue/.env

# 2. Отредактируйте пароли в backend/.env

# 3. Запустите (для разработки)
docker compose up -d --build

# ИЛИ Запустите (для продакшена с Nginx)
docker compose -f docker-compose.prod.yml up -d --build
```

Приложение доступно:
- Фронтенд: http://localhost:5173
- Бэкенд API: http://localhost:8080

### Вариант 2: Без Docker

**Бэкенд:**
```bash
cd backend
# Убедитесь, что .env файл настроен для вашей локальной БД
php -S localhost:8081 main.php
```

**Фронтенд:**
```bash
cd fronted-vue
pnpm install
pnpm run dev
```

## API Documentation

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/slots` | Получить все слоты (для администратора) или только доступные (для клиента) |
| GET | `/stats` | Получить статистику (только администратор) |
| GET | `/slots/{id}` | Получить конкретный слот |
| POST | `/auth/login` | Вход администратора |
| POST | `/auth/logout` | Выход администратора |
| POST | `/slots/generate` | Сгенерировать слоты по шаблону (только администратор) |
| POST | `/slots` | Создать один слот (только администратор) |
| POST | `/slots/{id}/book` | Забронировать слот (для клиента) |
| POST | `/slots/{id}/cancel` | Отменить бронирование (только администратор) |
| PUT/PATCH | `/slots/{id}` | Обновить описание слота (только администратор) |
| DELETE | `/slots/{id}` | Удалить слот (только администратор) |

## Переменные окружения

**backend/.env:**
- `DB_HOST` - хост MySQL
- `DB_PORT` - порт MySQL
- `DB_DATABASE` - имя БД
- `DB_USERNAME` - пользователь БД
- `DB_PASSWORD` - пароль БД
- `MASTER_PASSWORD` - пароль администратора (хэшируется при первом входе)
- `APP_ENV` - окружение
- `APP_DEBUG` - режим отладки (true/false)

**fronted-vue/.env:**
- `VITE_API_BASE_URL` - URL бэкенда (по умолчанию http://localhost:8080)

## Известные ограничения

- Отсутствует стандартный тестовый фреймворк для PHP (например, PHPUnit). Юнит-тесты реализованы в виде отдельных PHP-скриптов в директории `backend/tests/`.
- Фронтенд-тесты используют нативный Node.js test runner (`node:test`).
- Backend API не использует и не требует проверки CSRF-токенов.
- Состояние авторизации на фронтенде опирается на флаг `is_logged_in` в localStorage.
- CORS строго ограничен в `backend/main.php` (разрешены только конкретные домены, например `http://localhost:5173`).

## Безопасность

- Пароли хэшируются через Argon2id и хранятся в БД.
- Для взаимодействия с БД используются подготовленные запросы (prepared statements) `mysqli` для защиты от SQL-инъекций.
- В ответах API для не-администраторов вручную удаляются PII-данные клиентов (имя и телефон).
- Сессионная авторизация с защищенными куки (HttpOnly, SameSite).
- Rate limiting: блокировка IP после 5 неудачных попыток на 15 минут.
- Валидация входных данных и защита от пересечения слотов по времени.
